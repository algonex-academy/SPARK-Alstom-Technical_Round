1. Design a Ticket Management System.
-- DDL Commands
-- Creating Users Table
create table users (
    user_id int auto_increment Primary Key,
    name varchar(100),
    email varchar(100) unique
);

-- Creating tickets Table
create table tickets (
     ticket_id varchar(20) primary key,
     ticket_cost decimal(4,2),
     description text,
     user_id int,
     dept_id int,
     foreign key (user_id) references users(user_id),
     foreign key (user_id) references departments(user_id)

);
     
-- Creating Departments Table
create table departments (
     dept_id int Primary Key,
     dept_name varchar(20)
);

select t.ticket_id, t.description, u.name AS user_name, d.dept_name
from tickets t
join users u ON t.user_id = u.user_id
LEFT JOIN departments d ON t.dept_id = d.dept_id;

 -- Tickets without Department
select * 
from tickets
where dept_id IS NULL;


-- Users with Multiple Tickets
SELECT u.user_id, u.name C OUNT(t.ticket_id) AS total_tickets
FROM users u
JOIN tickets t ON u.user_id = t.user_id
GROUP BY u.user_id, u.name
HAVING COUNT(t.ticket_id) > 1;

 7. What is the difference between = vs LIKE
-> = is used for exact matching and the value must be 100% the same
    ex: select * from users where name = 'Kiran'
        -- the above query is gets only rows where name is exactly "Kiran"
-> LIKE is Used for searching patterns and works with wildcards which are % and _'
              here, % → any number of characters
                    _ → single character

10. what is Lambda Function and list comprehension, give an example for each
-> The Lambda function is a small, one-line function without a name, it is used when you need a function for a short time.
   Syntax: lambda arguments: expression
    ex: square = lambda x: x * x
        print(square(5)) # O/P is 25

 -> The list comprehension is the creating a short list for single lime code.
    Syntax: [expression for item in condition]
    ex: numbers = [1, 2, 3, 4]
        squares = [x * x for x in numbers]
        print(squares) # O/is 1, 4, 9, 16
