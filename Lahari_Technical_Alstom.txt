2. Why using VARCHAR for ticket_id is a bad idea?
If we use VARCHAR for ticket_id then different size of numbers are created while booking the tickets. So instead of VARCHAR if we use CHAR(size) then it allows to create fixed size of ticket_id for all the users.
So, By that it reduce the redundancy of creating the ticket_id.

-> Users with Multiple Tickets
select u.user_id, count(t.ticket_id) from users u
join ticket t
on u.user_id  = t.user_id
group by u.user_id
having count(t.ticket_id) > 1; -- finding users who have more than one ticket

-- Views 
Viwes is a virtual query stored table. It doesn't store the data physically. Once the mysql workbench is closed it will automatically dissaperared.
there are mainly 3 types of views - simple view, complex view, updatable view
syntax - 
create view view_name AS
select column1, column2
from table_name
where condition;

-- CTE(Common Table Expression)
A CTE is a temporary result set used within a query.
WITH cte_name AS (
    select column1, column2
    from table_name
    where condition
)
select *from cte_name;

-- Subquery types
subquery means query within a query
there are mainly 3 types of subqueries :
1.single row subquery - In this inner query returns only single row as output and send it to the outer query
syntax - 
select col1, (select avg(col) from tb_name)  from tb_name;
2.Multi row subquery - In this inner query returns multiple rows as output and ssend it to the output query
syntax-
select col1, (select col1, count(col) from tb_name
                group by col1) from tb_name;
3.Co-related subquery - In this the inner query depends on outer query. It iterates for every value. And it also works as self join
and there are many other forms like subquery with SELECT clause, subquery with FROM clause etc
