CREATE TABLE Tickets(user_id INT,
                     dept_id INT,
                     ticket_no VARCHAR(50) UNIQUE,
                     FOREIGN KEY (user_id) REFERENCES Users(user_id),
                     FOREIGN KEY (dept_id) REFERENCES Departments(dept_id)
                    );
-- Department table
CREATE TABLE Departments(dept_id INT PRIMARY KEY,
                         dept_name VARCHAR(50),

2.Why using VARCHAR for ticket_id is a bad idea?
=>Using VARCHAR for ticket_id is generally a bad idea because:
Performance: Numeric types (INT, BIGINT) are faster for indexing, searching, and sorting compared to strings.
Storage: VARCHAR consumes more space than numeric types for the same values.
Data integrity: Numeric IDs ensure consistent, sequential, and unique values, whereas strings can have formatting errors or accidental duplicates.
Arithmetic operations: It is easier to generate, increment, or compare numeric IDs than string IDs.

7.What is the difference between = vs LIKE
=>The = operator is used for exact matching, whereas the LIKE operator is used for pattern matching in SQL. 
When we use =, the value must match exactly as given. In contrast, LIKE allows searching using patterns 
with wildcards such as % (any number of characters) and _ (single character).
=>SELECT * FROM employees
WHERE name = 'Rani'; --This returns only rows where the name is exactly Rani.
=>SELECT * FROM employees
WHERE name LIKE 'R%'; -- This returns all names starting with R, such as Rani, Ram.

It is more readable and efficient than using loops.
=>ex:numbers = [1, 2, 3, 4]

11.Write a code for factorial & even/odd of a number within a single function
def check_number(n):
    if n % 2 == 0:
        print("Even")
    else:
        print("Odd")
    fact = 1
    for i in range(1, n + 1):
        fact *= i
    print("Factorial:", fact)
check_number(5)
